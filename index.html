WORKSHOP MAP

學員動作

先建立整體地圖。

STEP 03

它不只是聊天 AI

聊天 AI：一問一答

Claude Code：在你電腦裡面工作的 AI agent

講者重點

聊天 AI 像臨時顧問。Claude Code 比較像走到你的旁邊，它可以看資料夾、讀規則、建立檔案、修改檔案、連接更多App。

學員動作

說出自己過去用聊天 AI 最常卡住的地方。

STEP 04

Claude Code 能做什麼

  • 能操控你的電腦：讀檔、改檔、整理資料夾
  • CLAUDE.md 會被記住，不必每次重講
  • 用 Skill 學會你的專業流程，可重複又穩定
  • 連接各種工具
講者重點

對知識工作者來說，Claude Code 的價值不只是寫程式，而是處理結構化文字工作。

學員動作

理解它可以處理文章、教案、摘要、會議記錄、簡報腳本、專案資料。

RUN 05

安裝 Claude Code 及 VsCode

請依據 Anthropic及 VsCode 官方網站安裝

  • 最難的一關
講者重點

現場補充。

學員動作

先用共同範例資料練習，避免每個人一開始都卡在自己的資料太亂。

  • 會 Git：使用 git clone
  • 不熟 Git：GitHub 頁面按 Code -> Download ZIP
RUN 07

安裝前先理解一件事

工具不是課程核心。

不要背指令，要思考你如何整理資訊。

講者重點

Claude Code 的入口比較硬，尤其是命令列。這不代表它不適合知識工作者，只代表第一步需要有人帶。

學員動作

知道安裝指令要以上課當天官方文件為準。

RUN 08

第一次進入資料夾

進入範例知識庫：

cd llm-knowledge-base
claude
講者重點

你在哪個資料夾啟動 Claude Code，它就在哪個資料夾內工作，不會到其他資料夾。

學員動作

進入範例 repo，啟動 Claude Code。

STEP 09

我的工作畫面

建議工作畫面：

  • 左邊：檔案總管 / VS Code
  • 右邊：Claude Code
  • 中間：筆記或程式
講者重點

Claude Code 不是孤立聊天室，它最好和檔案視窗一起使用，看的到檔案怎麼變更。

學員動作

調整自己的工作畫面。

RUN 10

先熟悉互動：

  • Shift + Tab 切換模式

這一段先不要做大任務，只讓學員習慣和 Claude Code 互動。

常用指令：

  • /init
  • /clear
  • /context
  • /model
  • /rewind
講者重點

這些指令是和 Claude Code 協作時常用的控制鈕。

學員動作

逐一試用或觀看講者示範。

RUN 12

`/init` 是什麼

/init

讓 Claude Code 認識這個資料夾。

講者重點

這通常會產生或協助建立專案說明，讓 AI 知道這個資料夾的用途。

學員動作

在範例 repo 執行 /init，觀察 Claude Code 如何理解資料夾。

RUN 13

`/clear` 是什麼

/clear

清掉當前對話，避免脈絡混亂。

講者重點

當對話太長、任務換方向、AI 開始混亂時，可以重新開始。

學員動作

練習清掉對話後重新提出任務。

RUN 14

`/context` 是什麼

/context

可以即時查看 context window 使用情況。

講者重點

AI 不是無限記憶。知道它正在看什麼，才能知道為什麼它回答得好或不好。

學員動作

執行 /context，觀察 context 使用情況。

STEP 15

Markdown 是 AI 的母語

  • 純文字檔，副檔名 .md
  • 我幾乎把所有資料都轉成 .md
  • Markdown 只要先懂三個符號：
# 標題
- 清單
**重點**
講者重點

Markdown 是純文字，容易保存、搜尋、版本控制，也容易被 AI 理解。

學員動作

建立或打開 README.md，看懂基本格式。

STEP 16

CLAUDE.md 是 給AI的使用說明書

CLAUDE.md

不只是設定檔，更是給AI的說明書。

它告訴 AI：

  • 我是誰
  • 怎麼工作
  • 資料怎麼放
  • 什麼不能動
  • 怎樣算做得好
講者重點

如果沒有說明書，AI 只能猜你的偏好和規則。

學員動作

打開範例 repo 的 CLAUDE.md。

STEP 17

寫 CLAUDE.md 的四個原則

四個原則：

  1. 不要太長
  2. 隨錯誤迭代
  3. 模組化
  4. 和 Claude Code 一起寫
講者重點

CLAUDE.md 不是一次寫完的文件，而是協作中長出來的記憶。

學員動作

找出範例 CLAUDE.md 中最重要的三條規則。

STEP 18

先讓 AI 知道：

  • 我的身分

  • 我的寫作風格
  • 我的常見產出
講者重點

第二大腦不是抽象資料庫，而是服務特定人的知識系統。

學員動作

寫下自己的 5 條個人脈絡。

RUN 19

將資料匯入

  • 將雲端硬碟或Facebook貼文下載(哪些東西不應該放進來？)
  • 用 pandoc 或 markitdown 將檔案轉成 md
  • 直接叫Claude Code 轉成md
  • 用 /convert-to-md 把 EPUB / PDF / DOCX、甚至 Facebook 匯出的 JSON 一鍵轉成 Markdown
講者重點

試著請大家下載操作

學員動作

RUN 20

不要什麼都編譯

可以備份，不一定要編譯。

特別小心：

  • Facebook 舊貼文

  • 不再代表現在想法的資料（我真的想讓Claude知道我之前有多笨嗎？）
講者重點

備份可以很廣，編譯必須選擇。

學員動作

回答：哪些資料只該備份，不該進 wiki？

STEP 22

我的四層架構

資料夾規則：

  • raw：原始資料
  • wiki：編譯知識
  • brainstorming：思考探索
  • artifacts：對外成品
講者重點

資料夾規則不是收納術，而是在告訴 AI 每份資料現在扮演什麼角色。

學員動作

確認範例 repo 的資料夾結構。

STEP 23

第二大腦不是資料庫

第二大腦不是保存一切。

第二大腦是讓重要資料重新參與思考。

講者重點

如果只是把所有東西丟進資料夾，那叫備份，不叫第二大腦。

學員動作

區分「備份」和「編譯」。

NOTE 24

Git 是時光機

Git 是你的時光機。

它讓你知道：

  • 改了什麼
  • 何時改的
  • 必要時怎麼回去
講者重點

不需要把 Git 教成工程課，只要讓知識工作者知道它能保護自己的知識庫。

學員動作

看一次 git status 或由講者示範。

NOTE 25

Evaluation

沒有評估標準，AI 只能猜。

講者重點

品質標準是讓 AI 穩定的重要條件。

學員動作

NOTE 26

Context Engineering

AI 不是無限記憶。

Context Engineering就是幫 AI 看見該看的東西。

講者重點

資料夾階層架構、摘要、索引、命名規則，都是在幫 AI 降低 context 負擔。

學員動作

檢查自己的資料夾是否有清楚索引。

RUN 27

Skill 是什麼

Skill 是把你會重複做的事，變成可執行流程。

講者重點

Skill 不是魔法，而是把任務的步驟、規則、品質標準寫下來。

學員動作

RUN 28

Skill 不是越多越好

Skill 不是越多越好。

只把真的反覆需要的任務做成 Skill。

講者重點

太多 Skill 反而會讓系統混亂。

學員動作

STEP 31

ask-me：請 AI 先問清楚

ask-me

一次只問一個問題，直到共同理解。

範例
---
name: ask-me
description: 針對計畫或文章進行嚴謹訪談，直到達成共同理解。
---

請針對此想法的各個面向對我進行訪談。
一次只問一個問題。
如果問題可以透過探索資料夾回答，請先探索資料夾。
講者重點

很多 AI 失敗不是模型不夠好，而是太快開始。

學員動作

修改成自己的 ask-me 版本。
