PRINCIPLE

不是工具
是系統

工具會變，系統會留下。

PROBLEM

知識工作者不缺資料，缺的是再次使用資料的能力

資料很多

Drive、Notion、Obsidian、Line、PDF、貼文、會議記錄、課程靈感。

再次使用很難

找不到、連不起來、轉不成下一個作品。

資料不能被找回、連結、轉化，就只是倉庫。

SECOND BRAIN != DATABASE

第二大腦不是保存一切，
而是讓重要材料重新參與思考。

WHY CLAUDE CODE

它不是只能寫程式

它是能進入你的資料夾工作的 AI agent。

CHAT AI

CLAUDE CODE

進入工作室的夥伴

它能看資料夾、讀規則、建立檔案、累積工作流程。

CONTEXT ENGINEERING

DESIGN THE ROOM

設計 AI 看見什麼、忽略什麼、如何判斷

Prompt

你現在對 AI 說的一句話。

Context

它能看見的資料、規則、過去工作、品質標準。

MARKDOWN

Markdown 是 AI 的母語

純文字容易保存、搜尋、版本控制，也容易被 AI 理解。

# 標題 - 清單 **重點**
FOLDER AS THINKING

資料夾是一種思考設計

資料夾不是收納術。

資料夾是在告訴 AI：這份資料現在扮演什麼角色。

RUN TASK 01

建立第二大腦骨架

my-second-brain/ ├── raw/ ├── wiki/ ├── brainstorming/ ├── artifacts/ └── attachments/
CLAUDE.MD

合作契約

不是自傳，也不是願望清單。

MEMORY FILE

讓 AI 知道這個工作現場怎麼運作

我是誰

背景、工作、常見產出。

怎麼對話

語言、稱呼、提問規則。

資料怎麼放

資料夾結構與命名。

什麼不能動

raw 不覆蓋，避免破壞原始資料。

怎樣算好

品質標準與完成條件。

如何迭代

AI 做錯，就補規則。

RUN TASK 02

建立第一版 CLAUDE.md

請你先訪談我，協助我建立這個第二大腦資料夾的 CLAUDE.md。 一次只問一個問題。 請釐清我的身分、主要工作、常見產出、資料來源、 資料夾規則、命名慣例、AI 協作偏好與禁止事項。 訪談完成後，產出一份精簡、可維護的 CLAUDE.md。
DATA INTAKE

資料進系統前，先判斷來源

我寫的

代表我的想法與經驗。

別人寫的

需要保留來源脈絡。

AI 生成

避免污染自己的觀點。

RUN TASK 03

編譯第一份 wiki 摘要

請讀取 raw/ 中我指定的這份資料， 產出一份 wiki/summaries/ 摘要。 摘要請包含： 1. 原始資料路徑 2. 核心問題 3. 三到五個重點 4. 可發展的概念條目 5. 我未來可能如何使用這份資料 不要修改 raw/ 原始檔。
BRAINSTORMING

不要只保存答案，
也要保存你如何抵達答案。

RUN TASK 04

沉澱一次思考對話

請根據目前資料夾中的資料，和我討論以下問題： 「......」 請先提出你需要釐清的問題。 討論完成後，請把推理過程、關鍵結論、 未解問題與下一步，整理到 brainstorming/chat/。

SKILL

固定流程

TWO TYPES

Skill 不是越多越好

工作流 Skill

整理會議、編輯文章、建立學習專案、資料轉 Markdown。

領域知識 Skill

課程設計、論證審查、規格驅動、測試驅動。

它要來自你真的反覆需要的任務。

FIRST SKILL

ask-me：請 AI 先問清楚，再開始做

--- name: ask-me description: 針對計畫或文章進行嚴謹訪談，直到達成共同理解。 --- 請針對此想法的各個面向對我進行訪談。 一次只問一個問題。 如果問題可以透過探索資料夾回答，請先探索資料夾。
EVALUATION

沒有評估標準

GOOD SUMMARY

  • 關鍵概念

  • 未來用途
  • 不確定處標註假設
